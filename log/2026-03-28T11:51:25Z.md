**#1 (original):** A function counts the number of vowels in a string but returns a count that is
one too low; the bug is an off-by-one error in the loop range.
CRITIQUE: `for char in string` is idiomatic Python but cannot have an off-by-one error.
REPLACEMENT: Sliding-window function that misses the last window because the loop uses
`range(len(data) - k)` instead of `range(len(data) - k + 1)`.

**#2 (original):** A script calculates the average of a list of numbers but always prints zero;
the bug is integer division (`total / count` when both are `int`).
CRITIQUE: Python 3 treats `total / count` as float division.
REPLACEMENT: Script that accumulates exam scores with string concatenation instead of numeric
addition, producing a nonsensical total.

**#3 (original):** A function checks whether a number is even but always returns `True`; the bug
is using assignment (`=`) instead of comparison (`==`) inside the condition.
CRITIQUE: `=` cannot be used for assignment in a conditional expression in Python.
REPLACEMENT: Password validator that uses `and` instead of `or` to join two failure conditions,
so valid passwords are always rejected.

**#4 (original):** A temperature converter produces wrong results for some inputs; the bug is a

CRITIQUE: There is no natural way to write the Fahrenheit-to-Celsius conversion that does

REPLACEMENT: Degrees/minutes/seconds to decimal-degrees converter that divides seconds by 60
instead of 3600.
